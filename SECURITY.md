# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.4.x   | :white_check_mark: |

## Reporting a Vulnerability

Do not report security vulnerabilities through public GitHub issues.

### How to Report

1. Email `security@threatflux.ai`
2. Or use the repository Security tab to submit a private advisory

### What to Include

- Type of vulnerability
- Affected files, modules, or API surfaces
- Reproduction steps
- Expected impact
- Proof of concept when available

### Response Timeline

- Initial response: within 48 hours
- Status update: within 5 business days
- Resolution target: within 90 days

### Safe Harbor

We support good-faith security research that avoids privacy violations, data destruction, and service disruption, and that gives maintainers reasonable time to remediate before disclosure.
